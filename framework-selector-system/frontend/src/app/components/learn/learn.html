Understanding Agentic Frameworks

An agentic framework is a structured approach for designing AI-based applications that can independently accomplish goals in complex, changing environments.

The "Flight Booking" Test

Traditional AI

"Here are some tips and a link to search for flights from New York to London."

Agentic AI

Actually navigates the APIs, finds a flight under $600, and books it for you.

Core Aspects & Components

1

Planner / Reasoner

Turns high-level goals into ordered subtasks using planning algorithms or chain-of-thought. Richer planners give better control but cost more compute.

2

Memory & State

Retains prior decisions and environment state using short-term context windows or long-term external vector databases (like Pinecone or Chroma).

3

Tooling & Connectors

Connects the agent to the web, search, databases, code runtimes, or custom APIs, allowing it to take real-world actions.

4

Execution Orchestrator

Manages sequences, retries, parallel agent runs, and scheduling. In multi-agent setups, it handles communication and task allocation.

5

Multi-Agent Coordination

Roles, hierarchies, or swarms of agents (e.g., researcher, coder, QA) that collaborate or compete to solve complex tasks.

6

Safety & Guardrails

Enforces constraints, instruction filters, and human-in-the-loop checkpoints. Critical for preventing unauthorized or harmful actions.

Architectural Patterns

Reactive

Short loop: sense → immediate action. Used for fast, single-step decisioning (e.g., simple chatbots calling a single tool).

Pros: Low latency. Cons: Limited long-horizon planning.

Deliberative

Planner-centric. Agent builds explicit plans before acting (plan → execute → evaluate). Used for multi-step research or code generation.

Pros: Better for complex tasks. Cons: Slower, higher compute cost.

Hybrid

Fast reactive modules handle simple decisions, while a deliberative planner handles complex workflows. Common in production enterprise assistants.

Multi-Agent

Several specialized agents coordinate and pass messages either hierarchically or decentralized. Ideal for complex product development.

Evaluated Frameworks

LangGraph

Graph-based workflows

A stateful, multi-actor framework built on top of LangChain for robust agentic loops.

Key Differentiator: Stateful, deterministic execution

Selection Rationale: Production-grade state management

Microsoft AutoGen

Multi-agent conversation

Enables development of LLM applications using multiple conversing agents with a focus on enterprise scale.

Key Differentiator: Enterprise reliability patterns

Selection Rationale: Microsoft Research backing, conversation focus

CrewAI

Role-based teams

Framework for orchestrating role-playing, autonomous AI agents in a collaborative team structure.

Key Differentiator: Rapid prototyping, intuitive

Selection Rationale: Fastest-growing community, accessibility

LlamaIndex

RAG-first

A powerful data framework excelling at connecting custom data sources to large language models (RAG).

Key Differentiator: Document intelligence

Selection Rationale: Specialized RAG capabilities

Semantic Kernel

Enterprise SDK

Microsoft's SDK for seamlessly integrating AI capabilities and planner components into existing enterprise code.

Key Differentiator: Azure native, function composition

Selection Rationale: Microsoft enterprise integration

Haystack

Search pipelines

End-to-end framework for building custom NLP, retrieval, and search-heavy AI applications.

Key Differentiator: Production RAG optimization

Selection Rationale: Search-first architecture

OpenAI Swarm

Lightweight

An experimental framework focused on lightweight, highly controllable, and ergonomic multi-agent orchestration.

Key Differentiator: Educational simplicity

Selection Rationale: Learning-focused, minimal complexity

LangChain Agents

Comprehensive ecosystem

Standard agent implementations and tool integrations using the massive, community-driven LangChain library.

Key Differentiator: Breadth of integrations

Selection Rationale: Largest ecosystem, maturity

MetaGPT

SOP-based

Multi-agent framework that assigns structured roles (e.g., PM, Engineer, QA) to autonomously solve complex tasks.

Key Differentiator: Software development focus

Selection Rationale: Unique SOP paradigm

Strands Agents

AWS-native

A focused agentic framework specialized in strict workflow observability and high control.

Key Differentiator: Model-agnostic, OpenTelemetry

Selection Rationale: Cloud-native AWS integration

Current Limitations

  • Hallucinations: Agents can produce plausible but incorrect actions (e.g., sending wrong emails).
  • Safety Risks: Autonomous web/system actions raise compliance concerns without strict guardrails.
  • Cost & Compute: Planners making multiple LLM calls can become expensive at scale.
  • Explainability: Reconstructing the "why" of an autonomous decision is nontrivial.
  • "Agent Washing": Vendors labeling simple automation as "agentic" causes market confusion.

The Future

  • Standards & Evaluation: Better benchmarks and automated verification for agent behavior.
  • Human-Agent Collaboration: Mixed-initiative workflows with opt-in human approval.
  • Specialized Roles: Domain-specific agents (legal, medical) with certified knowledge.
  • Edge Agents: Efficient models running locally on devices for privacy.
  • Multi-Agent Marketplaces: Registries of certified, composable "agents as apps."
